The numeric restraint has hardware genres having everything to do with memory. In an abstract sense, we build memory starting with a bit, followed by a byte (being eight bits), and then we generalize from there with a type. A type being one or more bytes, assumed to have internal structure and procedures (class methods), but which are otherwise specific to that exact type. As for types, it is worthwhile splitting them up into word memory types which are less than or equal to word length (currently 64 bits or 8 bytes in laptops) in size. Next we have address memory types which are greater than the word length and less than or equal to two to the power of the word length (2^{word length})---at least in theory that is, in practice given current architectures it's often far less in size.
